# 📖 FarziPedia

### *de pictura mota — illuminata*

**Turn any YouTube video into a beautifully illustrated, Renaissance-codex-styled blog post.**

[Quick start](#quick-start) · [Deploy](#deploy) · [How it works](#how-it-works)

</div>

---

## What it does

You paste a YouTube URL. FarziPedia:

1. Fetches the captions
2. Downloads the video at low resolution
3. Samples frames at exactly the moments where a diagram, equation, slide, or other key visual appears (cue-driven + periodic baseline)
4. Hands the transcript and frames to Claude vision
5. Returns a structured blog post — title, sections, paragraphs, captioned images, callouts, key takeaways
6. Renders it as a magazine-style page with parchment background, Cinzel Roman caps, and Renaissance typography
7. Lets you download it as `.md` or save as PDF

**Faithful by design.** Every claim must be in the transcript or visible in a captured frame. No outside knowledge, no hallucination.

---

## Features

- **🎯 Smart screenshots** — frame extraction is keyed to visual cues in the transcript (*"as you can see"*, *"this diagram"*, color/position references), plus periodic baseline sampling. Adapts the frame count to video length: ~3 per minute, capped at 60.
- **🎨 Renaissance UI** — parchment background with subtle paper grain, a Vitruvian compass behind the wordmark masked to fade out in the centre, hand-drawn ink corner brackets on every card, and a wax-seal submit button.
- **⏳ Live ETA + topic-aware trivia** — the loading page shows an estimated time-to-completion derived from video duration, and rotates 14 trivia items related to the topic (generated by Claude Haiku in parallel with the video download).
- **📚 Faithful generation** — strict system prompt requires Claude to ground every claim in transcript or visible frames. Image references are constrained to provided timestamps.
- **💾 Two export formats** — Markdown via a server endpoint, or PDF via browser print-to-PDF with a tuned print stylesheet (A4, 18mm margins, no breaks inside figures).

## How it works

```
URL ─┬─► metadata (yt-dlp --skip-download)        ~3s
     │
     ├─► trivia generation (Haiku, background)    ~5s, parallel
     │
     ├─► transcript fetch                          ~2s
     │
     ├─► video download (yt-dlp 480p mp4)          15s–5m
     │
     ├─► frame extraction (ffmpeg, 4-way parallel) ~5s
     │
     └─► blog generation (Sonnet 4.5 vision)       30–60s
                       │
                       ▼
                  jobs/{id}/blog.json
                       │
                       ▼
                 rendered HTML page
```

## Quick start

Prerequisites: Python 3.11+, [ffmpeg](https://ffmpeg.org/download.html) on `PATH`, an [Anthropic API key](https://console.anthropic.com/settings/keys).

```bash
git clone https://github.com/FarziBuilder/farzipedia.git
cd farzipedia

pip install -r requirements.txt

# Bash / zsh
export ANTHROPIC_API_KEY=sk-ant-...

# PowerShell
$env:ANTHROPIC_API_KEY = "sk-ant-..."

python app.py
# open http://127.0.0.1:8000
```

## Deploy

> **YouTube blocks data-centre IPs.** Render, Fly, AWS, GCP — they all hit `RequestBlocked` from YouTube without a residential proxy. Webshare gives 10 free proxies; sign up at [webshare.io](https://www.webshare.io/), grab username + password, and set them as env vars (see [Configuration](#configuration)).

### Render (recommended)

1. Fork or push this repo to your GitHub.
2. [render.com](https://render.com) → **New → Blueprint** → connect repo. Render auto-discovers [`render.yaml`](render.yaml).
3. Set the three secret env vars in the dashboard:
   - `ANTHROPIC_API_KEY`
   - `WEBSHARE_PROXY_USERNAME`
   - `WEBSHARE_PROXY_PASSWORD`
4. Add your custom domain in **Settings → Custom Domains**, point a CNAME at the `*.onrender.com` value Render gives you. SSL auto-provisions.

### Fly.io

```bash
flyctl launch --copy-config --no-deploy
flyctl secrets set \
  ANTHROPIC_API_KEY="sk-ant-..." \
  WEBSHARE_PROXY_USERNAME="..." \
  WEBSHARE_PROXY_PASSWORD="..."
flyctl deploy
flyctl certs add farzipedia.yourdomain.com
```

### Docker (anywhere)

```bash
docker build -t farzipedia .
docker run -p 8000:8000 \
  -e ANTHROPIC_API_KEY=... \
  -e WEBSHARE_PROXY_USERNAME=... \
  -e WEBSHARE_PROXY_PASSWORD=... \
  farzipedia
```

## Architecture

```
farzipedia/
├── app.py            # FastAPI: routes, background jobs, MD download
├── pipeline.py       # Orchestrator: URL → frames → blog.json
├── transcript.py     # YouTube caption fetch (proxy-aware)
├── planner.py        # Capture-timestamp planning (cue + periodic)
├── analyzer.py       # Claude vision call → structured blog JSON
├── trivia.py         # Topic-aware loading-screen trivia (Haiku)
├── templates/
│   ├── base.html     # Fonts, palette, tailwind config
│   ├── index.html    # Landing page (Vitruvian centerpiece)
│   ├── job.html      # Loading page (hourglass + ETA + marginalia)
│   └── blog.html     # Final post (parchment, captioned figures)
├── static/style.css  # Parchment texture, ornaments, drop caps, print
├── Dockerfile        # Slim Python + ffmpeg + the app
├── render.yaml       # Render Blueprint config
├── fly.toml          # Fly.io config
└── requirements.txt
```

### How frames are picked

[`planner.py`](planner.py) reads the timestamped transcript and assembles a list of capture moments from three sources:

- **Cue phrases** — regex-matched speech that suggests the speaker is pointing at something (*"as you can see"*, *"this diagram"*, *"on the left"*, color/position references). Captures fire ~1.5 s after the cue.
- **Periodic baseline** — every 30 s, regardless of cues, so passive slide changes get caught.
- **Mandatory** — an early frame for the title card, the first frame after any silence > 8 s, a frame near the end if a recap is signaled.

Candidates are deduped within 6 s, then capped at `min(60, max(20, duration / 60 × 3))` to keep Claude's input under ~100 images.

### How the blog is generated

[`analyzer.py`](analyzer.py) sends a single message to `claude-sonnet-4-5` with all the frames inline (each labeled with its timestamp), the full transcript, and a strict system prompt. Claude returns a single JSON object: `title`, `subtitle`, `hero_timestamp`, `sections[].blocks[]` (paragraph, image, callout, code, quote), `key_takeaways[]`. The model picks which image timestamps to reference; unreferenced frames are silently dropped from the rendered page.

## Limitations

- **In-memory job state.** The job dictionary lives in process memory; restarting the server forgets jobs in progress. Completed blogs persist as `jobs/{id}/blog.json` on disk — mount a persistent volume to keep them across deploys.
- **No auth or rate limiting.** Single-tenant app. Front it with nginx, Cloudflare, or Caddy if you expose it.
- **Cost scales with video length.** A 15-min video costs ~\$0.05–\$0.15 in Claude vision tokens; a 60-min lecture is ~\$0.30–\$0.60.
- **Caption-disabled videos won't work.** The transcript is the source of truth for the writing; the app fails fast with a clear error.
- **yt-dlp drift.** YouTube frequently changes streaming protocols. If downloads start failing, redeploy — Docker rebuilds pull the latest `yt-dlp` from `requirements.txt`.

## Tech stack

- **Backend**: FastAPI, Jinja2, threading
- **Models**: Claude Sonnet 4.5 (vision, blog generation), Claude Haiku 4.5 (trivia)
- **Video**: [yt-dlp](https://github.com/yt-dlp/yt-dlp) (≤480p mp4), [FFmpeg](https://ffmpeg.org/) (frame extraction)
- **Captions**: [youtube-transcript-api](https://github.com/jdepoix/youtube-transcript-api)
- **Frontend**: Tailwind via CDN, vanilla JS, custom CSS for parchment + ornaments + print stylesheet
- **Fonts**: [Cinzel](https://fonts.google.com/specimen/Cinzel) (Roman caps), [Cormorant Garamond](https://fonts.google.com/specimen/Cormorant+Garamond) (display serif), [IM Fell English](https://fonts.google.com/specimen/IM+Fell+English) (old-book body), [Italianno](https://fonts.google.com/specimen/Italianno) (script flourishes)
